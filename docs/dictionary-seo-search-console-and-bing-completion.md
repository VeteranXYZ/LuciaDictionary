## Sitewide SEO Enhancement

Completed on 2026-07-15 for the same production origin.

Implemented signals:

- Centralized site identity in `src/lib/seo.js`: production origin, site name, alternate names, logo URL, and reusable structured-data builders.
- Added global `Organization` JSON-LD on every rendered page.
- Added richer homepage `WebSite` and `WebApplication` JSON-LD, including bilingual language signals, educational use, audience, and feature list.
- Removed the previous homepage `SearchAction` JSON-LD because the app does not expose a stable crawlable search-result URL.
- Removed same-URL `hreflang` declarations after review. The site has one bilingual URL per page, not separate locale-specific URL versions; `hreflang` should be introduced only if distinct localized routes are added later.
- Added Open Graph locale metadata and fuller Twitter card metadata.
- Added visible crawlable homepage summary copy and public links to the product information pages.
- Added visible breadcrumb navigation plus `WebPage` and `BreadcrumbList` JSON-LD on the four information pages.
- Added a visible, ordered guide on `/guide/` with standard `WebPage` and `BreadcrumbList` structured data.
- Expanded content on `/about/`, `/sources/`, and `/guide/`; the latter includes the former standalone accessibility guidance.
- Expanded `npm run audit:seo` to enforce the exact canonical URL set, the absence of redirect rules, the absence of unsupported same-URL `hreflang` and SearchAction markup, social metadata, Organization JSON-LD, info-page structured data, homepage WebApplication data, GA4 page-view configuration, and Privacy-page GA4 disclosure.

Deliberately not implemented:

- No autogenerated `/word/*` pages were created. The current app does not yet have dedicated, unique, high-quality public word-entry pages, so generating thin crawlable word URLs would be a quality risk.
- No crawlable `/search` pages were added. Search remains an interactive client-side workflow and `/search` returns `404`.
- No unsupported rich-result schema was added. Structured data is limited to page identity, breadcrumbs, and product/app information.
