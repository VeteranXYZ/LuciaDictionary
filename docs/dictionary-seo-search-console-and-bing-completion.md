## Sitewide SEO Enhancement

Completed on 2026-07-15 for the same production origin.

Implemented signals:

- Centralized site identity in `src/lib/seo.js`: production origin, site name, alternate names, logo URL, and reusable structured-data builders.
- Added global `Organization` JSON-LD on every rendered page.
- Added richer homepage `WebSite` and `WebApplication` JSON-LD, including bilingual language signals, educational use, audience, and feature list.
- Removed the previous homepage `SearchAction` JSON-LD because the app does not expose a stable crawlable search-result URL.
- Added self-referencing `hreflang` alternates for `x-default`, `zh-CN`, and `en` on the six canonical pages.
- Added Open Graph locale metadata and fuller Twitter card metadata.
- Added visible crawlable homepage summary copy and public links to the product information pages.
- Added visible breadcrumb navigation plus `WebPage` and `BreadcrumbList` JSON-LD on the five information pages.
- Added `HowTo` JSON-LD on `/how-to/` using the same visible steps shown on the page.
- Expanded content on `/about/`, `/sources/`, and `/accessibility/` so each indexable information page has more specific, useful product context.
- Expanded `npm run audit:seo` to enforce canonical URLs, hreflang, social metadata, Organization JSON-LD, info-page structured data, homepage WebApplication data, and the absence of unsupported SearchAction markup.

Deliberately not implemented:

- No autogenerated `/word/*` pages were created. The current app does not yet have dedicated, unique, high-quality public word-entry pages, so generating thin crawlable word URLs would be a quality risk.
- No crawlable `/search` pages were added. Search remains an interactive client-side workflow and `/search` continues to canonicalize to `/`.
- No unsupported rich-result schema was added. Structured data is limited to page identity, breadcrumbs, product/app information, and the how-to instructions that are visibly present.
